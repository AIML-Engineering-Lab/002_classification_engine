%% ── 1. PROJECT OVERVIEW ────────────────────────────────────────────────────
\section{Project Overview}

The primary objective of this project is to construct a robust binary classification pipeline capable of predicting outcomes across diverse domains, specifically focusing on the authenticity of ancient manuscripts and the pass/fail status of silicon chips during critical timing tests. The manuscript dataset features a heavy class imbalance, with only 20\% of the entries indicating authenticity, while the silicon chip dataset presents a relatively better balance of 71\% pass and 29\% fail. This disparity necessitated careful metric selection, as traditional accuracy measurements may obscure true model performance. The project builds upon foundational algorithms like the sigmoid function, logistic regression, and cross-entropy loss, demonstrating how to leverage advanced techniques including SMOTE for handling class imbalance, and L1/L2 regularization for feature selection and model robustness.

The approach is systematic, beginning with Exploratory Data Analysis (EDA) to gauge class distributions and feature relationships, followed by a preprocessing phase utilizing StandardScaler and OneHotEncoder. The project introduces logistic regression from first principles, illustrating how to compute probabilities via the sigmoid function and optimize model parameters using gradient descent techniques. Following a hands-on implementation, we leverage scikit-learn's built-in LogisticRegression model to enhance robustness through regulation. Key to this process is a comprehensive evaluation phase that encompasses accuracy, precision, recall, F1-score, AUC-ROC, and AUC-PR metrics, providing a nuanced understanding of model efficacy. Ultimately, the pipeline is validated across both datasets, affirming its versatility and generalizability.

\subsection{Key Concepts}

\textbf{Sigmoid Function:} Maps the predicted log-odds to probabilities, creating a decision boundary at σ(z) = 0.5, allowing for interpretation of continuous outputs as binary classifications.

\textbf{Cross-Entropy Loss:} A metric specifically tailored for binary classifications that emphasizes penalizing confident wrong predictions more heavily than minor misclassifications, ensuring stable gradient descent optimization.

\textbf{SMOTE:} A technique employed to address class imbalance by generating synthetic minority class samples, enabling a more balanced training dataset and improving model performance.

\subsection{Processing Pipeline}

The processing pipeline begins with data loading, where datasets are ingested and structured for analysis. A thorough Exploratory Data Analysis (EDA) is then performed to understand distributions, reveal class imbalances, and identify potential correlations between features and target labels. This stage is crucial as it informs the subsequent preprocessing decisions, which include normalizing certain features via StandardScaler and applying OneHotEncoder where categorical variables are present, thus ensuring the model can effectively interpret the data.

Next, we tackle class imbalance directly by employing SMOTE to augment the minority class within the manuscript dataset. This step is vital to enhance the model's predictive power and reduce bias towards the majority class. Following data preparation, logistic regression is implemented from first principles, allowing us to visualize the decision boundary derived from the sigmoid function and understand model behavior through the lens of cross-entropy loss. The transition to utilizing scikit-learn's LogisticRegression library occurs here, wherein we experiment with L1 and L2 regularization methods to optimize model complexity and performance.

The evaluation phase encompasses calculating a comprehensive suite of metrics that are essential for binary classification tasks, including accuracy, precision, recall, F1 score, AUC-ROC, AUC-PR, and the Matthews Correlation Coefficient (MCC). These metrics guide us in comparing model performances, ensuring that a variety of angles are covered in terms of evaluation. Lastly, probability calibration is performed using Platt Scaling, leading to a reliability diagram that visually represents model confidence against actual outcomes. This final evaluation helps to ascertain the effectiveness and reliability of our classification pipeline across both domains.

%% ── 4. RESULTS ─────────────────────────────────────────────────────────────
\section{Results}

Upon completing the evaluation, the results reveal that the models effectively handle the dataset nuances, particularly the class imbalance in the manuscript authenticity dataset through the application of SMOTE and selected regularization techniques. The logistic regression from scratch method provided valuable insights regarding the optimization process, while the scikit-learn implementation yielded more robust performance metrics across the validation set. The accuracy on the silicon dataset was notably high, reflecting well on the efficacy of the predictive modeling. However, the manuscript dataset's accuracy was suppressed by severe class imbalance, highlighting the importance of the metrics employed beyond accuracy.

Additionally, threshold analysis illuminated the importance of selecting an appropriate classification threshold rather than relying on the default value of 0.5. By adjusting the threshold based on the cost of false positives and negatives, we can reduce misclassification in critical applications such as document verification and chip validation. Overall, the project illustrates a successful aim to bridge cross-domain algorithm application while addressing real-world imbalances in classification tasks.
