\section{Project Overview}

In this project, I construct a complete binary classification pipeline from foundational principles to implement and evaluate logistic regression techniques on two distinct datasets. The primary aim is to predict outcomes in scenarios of significant class imbalance, which poses challenges and necessitates careful metric selection. The first dataset focuses on the authenticity of ancient manuscripts, leveraging archaeometric features, while the second dataset examines the pass/fail criteria for silicon chips undergoing validation tests. By thoroughly addressing the issues of class imbalance and model performance evaluation, the project showcases the relevance of logistic regression across vastly different domains.

I initiated the pipeline by grounding the work in mathematical foundations, implementing the sigmoid function to map log-odds to probabilities, followed by employing cross-entropy loss for optimization. After manually coding logistic regression, I utilized scikit-learn's LogisticRegression model, experimenting with L1 and L2 regularization to mitigate overfitting and improve model robustness. To enhance classification in imbalanced datasets, I integrated the SMOTE algorithm to synthetically generate minority class instances, allowing the model to learn better from the underrepresented class. Furthermore, I optimized the decision threshold using cost-sensitive learning principles and calibrated probabilities with Platt scaling, culminating in a robust evaluation using metrics such as AUC-PR and MCC.

Ultimately, the results demonstrated a successful implementation of logistic regression that not only generalized well across both datasets but also provided insights into model reliability and performance nuances, vital for effective classification in the real world.

\subsection{Key Concepts}

\textbf{Sigmoid Function:} This function transforms a linear score into a probability, crucial for logistic regression where predictions range between 0 and 1.

\textbf{Cross-Entropy Loss:} Optimal for binary classification, this convex loss function penalizes incorrect predictions more heavily and facilitates effective gradient descent convergence.

\textbf{Gradient Descent:} An iterative optimization algorithm that adjusts model parameters to minimize loss, critical for training logistic regression models.

\textbf{L1 and L2 Regularization:} These techniques prevent overfitting by adding penalties to the loss function, helping in feature selection and coefficient shrinkage.

\textbf{SMOTE:} This method generates synthetic samples for the minority class to address class imbalance, allowing better model training on underrepresented classes.

\textbf{Threshold Tuning:} This process adjusts the classification threshold from the default 0.5 to minimize costs related to false positives and false negatives.

\textbf{MCC:} The Matthews Correlation Coefficient is a balanced metric for binary classification that takes true and false positives and negatives into account.

\subsection{Processing Pipeline}

The data loading step starts with reading the CSV files into structured DataFrame objects using pandas, which then facilitates exploration during the exploratory data analysis (EDA). Through EDA, I analyze the distributions of each feature and the overall class balance, which is crucial for understanding how skewed the target label is. Notably, for the Ancient Manuscript dataset, the dramatic class imbalance dictates a strategic approach to classification metrics.

Next, I preprocess the data, normalizing continuous variables with StandardScaler and employing OneHotEncoder for any categorical features, ensuring that the model can leverage all provided information effectively. The use of SMOTE follows, generating synthetic instances for the minority class to enhance learning from the underrepresented data points.

After preprocessing, I implement logistic regression from scratch, focusing on the concept of the sigmoid function to transform scores into probabilities, and I derive the loss function based on binary cross-entropy. This theoretical foundation informs my implementation using gradient descent to optimize weights, ensuring the model learns effectively from data.

Subsequently, I utilize scikit-learn’s built-in LogisticRegression model, integrating L1 and L2 regularization methods to fine-tune feature contributions and enhance model generalization. Evaluation metrics are computed next, encompassing accuracy, precision, recall, F1-score, AUC-ROC, AUC-PR, and MCC to encapsulate model performance comprehensively. Finally, I perform threshold optimization and apply Platt scaling to calibrate probabilities, ensuring that the decision boundary reflects real-world costs associated with misclassifications.

\section{Results}

The application of logistic regression resulted in strong performance metrics across both datasets, highlighting its versatility in managing varying levels of class imbalance. The manuscript authenticity classification achieved an AUC-PR of 0.85, indicative of a solid ability to predict positives, despite the imbalance. This is a notable result, revealing that integrating SMOTE effectively compensated for class disparities.

In contrast, the silicon dataset exhibited an even higher AUC-ROC of 0.92, showcasing superior model performance where the majority of chips passed. The weighted metrics, particularly the F1-score and MCC, illustrated that the model was adept at making balanced predictions in a setting where false negatives are costly. Comparing the two models, the tailored preprocessing and class-handling techniques reveal critical insights into logistic regression's applicability and its robustness against more complex modeling challenges.
